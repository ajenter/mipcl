Compilation and linking

If in a source C++ file there are calls to any MIPCL functions, this file must include one of the following lines: 

 #include "CMIP.h" // if a MIP is solved
 #include "CLP.h" // if an LP is solved

MIPCL can be used on any computer platform that has a standard C++-compiler. In this section we show how to translate and link MIPCL programs by using two widely used compilers: g++ (GNU-compiler) and Visual C++.

GNU compiler

The GNU compiler is used practically on all platforms (Unix, Linux, Windows and many others). When developing complex C++-applications, we usually use the utility make that reads instructions from a scenaio file usually named makefile. Such a file contains instructions that tells make how to transform source files into the executable program.

Here we are not going to teach writing makefiles. A standard advice is, if you are a beginner at writing such files, it is better to start from an existing template file and adjust it to your needs. Many examples of makefiles written for the aplications distributed together with the MIPCL are in the folder 

 $MIPCLDIR/examples/mipcl

where `MIPCLDIR is an environment variable that points to the MIPCL installation directory.

All makefiles supplied with MIPCL will mostly work with different versions of make, however, there tends to be a few incompatibilities, so we recommend installing GNU make if you don't already have it on your system and using it. (It may be called gmake rather than make.)

You may also need to set the LD_LIBRARY_PATH environment variable (see Installation of MIPCL) so the system dynamic linker can find newly installed libraries.

For those who is familiar with makefiles, it is enough to explain how to compile and compose simple MIPCL applications, which is explained in Primer: compilation and linking.

Visual C++

As a rule, the Visual C++ compiler is used within Microsoft Visual Studio .Net. After creating a Visual Studio project for your application, modify it by the following schema:

  • project → Properties → Configuration → All Configurations:
    • C/C++ → General → Additional Include Directories: $(MIPCLDIR)/mipcl/headers
    • Linker → General → Additional Library Directories: $(MIPCLDIR)/mipcl/bin
    • Linker → Input → Additional Dependencies: mipcl.lib

Remark 1. If your project configurations (say, Debug and Release) differ in modified filds, the above actions are to be repeated for each configuration.

Remark 2. You can follow the same scheme in case you are using the INTEL-compiler that are integrated into Visual Studio.
