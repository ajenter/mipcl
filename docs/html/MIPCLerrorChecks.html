Checking programming errors

The MIPCL does not check for programming errors such as passing wrong pointers, or illegal argument types or values.

The MIPCL does not check for these programming errors for the following reasons:

  • forcing MIPCL functions to check all possible error conditions degrades the performance of correct methods;
  • in many cases, there is not enough runtime information to perform such checking;
  • forcing MIPCL functions to check for all possible error conditions would likely result in such checks to be duplicated – once in the user code, and then again in the library.

It worth noting, that most C library functions do not guard programming errors.
