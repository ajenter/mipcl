Using MIPCL in GUI applications

Run time messages

When solving a MIP or an LP, MIPCL displays messages to inform users about current states of the solution process. These messages are sent to the standard output stream, or to the standard error stream. Such a behavior is not appropriate for applications with graphic user interface (GUI).

When solving simple problems (that are solved in short time), we can simply call the function

void CLP::beSilent(bool flag);

with flag set to true to block all run time messages. To resume printing run time messages, call beSilent(false).

But when solving complex problems that require lengthy computations, switching off information messages is undesirable. A better solution is to overload the next six virtual functions:

void CLP::infoMessage();
void CLP::lpInfo();
void CMIP::mipInfo();
void CMIP::cutInfo();
void CMIP::probingInfo();
void CMIP::solStatistics().

Since solStatistics() is also used for writing solution statistics into a text file, in GUI applications it should be overloaded with some care. A useful template for overloading solStatistics() is as follows:

void CDerivedMIP::solStatistics(std::ostream &out, const char* MIPCLver,
const char* solTime, bool timeLimit, int nodeNum,
bool feasible, bool hasSolution, double objVal,
bool opt, double gap, bool gapLimit, double bound, int difficultNodes)
{
if (out == cout) {
// TODO: write your code here
}
else
CMIP::solStatistics(out,MIPCLver,solTime,timeLimit,nodeNum,
feasible,hasSolution,objVal,opt,gap,gapLimit,bound,difficultNodes);
}
